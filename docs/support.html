Frequently Asked Questions

How do I enroll a child's device?
From the parent's device, open Big Brother and navigate to the family management screen. Generate an invite code for the child. On the child's device, install Big Brother, select "Join as Child," and enter the invite code. The device will be enrolled in your family and ready to receive commands.
What are the different lock modes?
Big Brother has four modes:

Unlocked — No restrictions. The device operates normally.
Locked — Only the apps you've marked as "daily allowed" are accessible. Everything else is blocked.
Essential Only — Only critical system apps (Phone, Messages, etc.) remain available.
Disabled — All non-essential functionality is blocked. The device can still receive calls and emergency services.
How does Temporary Unlock work?
Parents can grant a time-limited unlock from their device. You set the duration (e.g., 30 minutes for homework), and the child's device automatically returns to its previous lock mode when the timer expires. No action needed from the parent to re-lock.
What happens if FamilyControls permission is revoked?
If a child revokes the FamilyControls permission in Settings, the App detects this through its authorization health monitoring system. The parent will be notified that the device's protection status has changed. The child will need to re-grant permission, or the parent can re-enroll the device. Note: on supervised devices, this permission cannot be revoked by the child.
Does the App require Apple Family Sharing?
No. Big Brother uses its own enrollment system with invite codes. It works regardless of whether devices share an Apple ID, are in a Family Sharing group, or are completely independent accounts. This makes it ideal for larger families or non-traditional setups where Apple's built-in limits are too restrictive.
Where is my data stored?
All family data is stored in Apple CloudKit within your own iCloud account. No data is sent to third-party servers. PINs are hashed and stored locally in the device Keychain. See our Privacy Policy for full details.
What if a device goes offline?
The heartbeat system will detect that the device has stopped checking in. The parent dashboard shows the last known status and when the device was last seen. Any commands sent while the device is offline will be delivered as soon as it reconnects, thanks to the polling fallback system. Existing restrictions remain enforced even while offline.
Can multiple parents manage the same family?
Yes. Both parents can enroll as parent-role users in the same family. Each parent has full control over all enrolled child devices and can send commands independently.
How quickly do commands take effect?
Commands are delivered via push notifications and typically take effect within seconds. If push delivery fails, the device's polling system will pick up the command within a short interval as a fallback.
